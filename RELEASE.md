# Release Checklist

Use this checklist before tagging or deploying LangelerMVC.

## Required Verification

1. Run `composer test`.
2. Run `composer verify:platform`.
3. Run `composer test:db-matrix` against available MySQL, PostgreSQL, and SQL Server services.
4. Run `composer ops:health` and confirm readiness checks match the target environment.
5. Exercise the installer from `Public/install/index.php` against the intended database, cache, session, queue, mail, payment, commerce, and admin-account settings.

## Production Preflight

- Confirm `.env` was generated by the installer or reviewed against `.env.example`.
- Confirm all secrets and runtime keys are deployment-local and not committed.
- Confirm `Storage/Secure`, `Storage/Cache`, `Storage/Logs`, uploads, sessions, and queue runtime paths are writable by the PHP process.
- Confirm migrations have run, including promotion, fulfillment, entitlement, subscription, order-state, and audit/queue tables.
- Confirm admin routes are protected by authentication and RBAC.
- Confirm payment and subscription provider credentials, webhook URLs, return URLs, and callback signatures are configured before live mode.
- Confirm carrier/shipping integrations are configured for the target region, especially Swedish carrier flows when enabled.

## Release Notes

- Update `CHANGELOG.md` with the tagged version and date.
- Update `Docs/FrameworkStatus.md` with the latest test count and verification date.
- Record any skipped matrix/runtime checks as release caveats.
